Making API Calls

Sending Requests

The ACTIVE.com Activity Search API uses simple HTTP GET requests and returns the information formatted in JSON.

First, let's take a look at the HTTP request format:

URL: http://api.amp.active.com/v2/search?{queryString params}&api_key={key}

Let's look at this piece-by-piece.

  • http://api.amp.active.com/v2/ - This is the base URL for the request. It will be needed for every request we are going to make.
  • search?{queryString params} - Here is will we will piece together what we what to look for. A glance at the options available in the documentation shows around 40 different options listed, but any of the attributes of the responses can be used as parameters. We will only be using a handful here, but you can see with the options available that you can make pretty specific searches.
  • & - The '&' symbol is used in query strings to separate the parameters being searched on.
  • api_key={key} - Here is where we will be using the key that we obtained in the earlier step, this is also going to be used in all of our requests.

How do we use this stuff?

The Basics

Here's the basic, and easiest, way to use this information:

  • First, we think about some things we want to search for...
    Let's say we're looking for triathlons, within 100 miles of Houston, TX.
  • Next, we substitute our criteria and key into our URL format above...
    http://api.amp.active.com/v2/search?query=triathlon&category=events&near=Houston,TX,US&radius=100&api_key=DH7XYYRXG5PNHB8RUHM23GPJ
